Where should uploaded images be stored?

The editor uses CodeMirror. When you paste an image, it needs to be uploaded somewhere and a URL returned to insert into the markdown. Which storage approach fits best?

A

Supabase Storage (recommended)

Add Supabase to playdeck (you already use it in the repo). Free tier includes 1GB storage. Simple REST API, public bucket for images, integrates with your existing Supabase project. Standard choice for Next.js apps.

Pros

  • Already in your stack (other apps use it)
  • Free tier generous
  • CDN-backed public URLs
  • Easy SDK

Cons

  • Need to add supabase client to playdeck
  • Requires Supabase project setup
B

Cloudflare R2

You're already using Cloudflare Workers MCP. R2 is S3-compatible object storage with zero egress fees. Great for images served globally. Requires a Cloudflare account + R2 bucket + API token.

Pros

  • Zero egress cost
  • Global edge CDN
  • S3-compatible API

Cons

  • New infrastructure to set up
  • Adds AWS S3 SDK dependency
C

Vercel Blob

Since you're likely deploying on Vercel, Vercel Blob is a first-party option. Minimal setup — just add @vercel/blob and a token. Files get a public CDN URL automatically.

Pros

  • Easiest setup for Vercel deployments
  • First-party SDK, no config
  • CDN included

Cons

  • Paid after 500MB
  • Vendor lock-in to Vercel
