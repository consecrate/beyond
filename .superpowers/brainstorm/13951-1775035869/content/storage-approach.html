How should the paste → render flow work?

Jazz has a first-party createImage() from jazz-tools/media that stores images as ImageDefinition CoValues with auto-resizing and blur placeholders — no external storage needed.

The image gets a Jazz CoValue ID like co_z123abc. The question is: what goes into the markdown, and how does the preview resolve it back to a visible image?

A

Store Jazz ID in markdown — resolve at render time (recommended)

Paste image → upload via createImage() → insert ![alt](jazz:co_z123abc) into markdown. A custom marked renderer intercepts jazz: URLs and resolves them to object URLs via FileStream.toBlob().

Pros

  • Fully in Jazz — syncs with deck ownership/permissions
  • Works offline (Jazz peer caches)
  • Clean markdown, easy to parse

Cons

  • Custom resolver needed in the marked pipeline
  • Async resolution requires a React hook or effect
B

Embed as data URL in markdown

Paste image → resize to reasonable size → convert to base64 → insert ![alt](data:image/png;base64,...) directly. No Jazz storage, no resolution step — it just works with the existing marked pipeline.

Pros

  • Zero infrastructure changes
  • Works immediately with existing render pipeline
  • No async resolution

Cons

  • Bloats markdown string (Jazz syncs the whole thing)
  • Large images = large CoValue payload
  • Not reusable across slides
