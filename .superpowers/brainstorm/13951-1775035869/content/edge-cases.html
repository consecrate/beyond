Edge cases — which ones should we handle in v1?

Select everything that needs to be handled. We'll spec each one explicitly so nothing is left to "we'll figure it out later".

📥 Input edge cases

1

Paste non-image (PDF, zip, text)

Clipboard contains a file that isn't an image. Should silently ignore — don't break normal text paste.

2

Image too large (>5MB / huge dimensions)

Jazz createImage() has maxSize option. Auto-resize to 1024px longest side before upload. No user prompt needed.

3

Paste/drop multiple images at once

e.g. select 3 files in Finder and drop all at once. Insert them sequentially on separate lines.

4

Drop a non-image file onto the editor

Should ignore the drop and not insert anything. Don't replace text at drop point.

5

Jazz upload fails mid-way

createImage() throws. Remove the ![uploading…]() placeholder and show a toast error.

6

Image added to deck while session is live

LiveSession.markdown is frozen at session start — new images won't appear in live session. Expected behaviour, but should be documented/noted.

🖼️ Render edge cases

7

Jazz image fails to load

FileStream.toBlob() returns undefined or throws. Show a grey broken-image placeholder with the alt text. Don't crash the slide.

8

Slow image load (Jazz still syncing)

Show blur placeholder (Jazz createImage generates one automatically) while full image loads. Progressive reveal.

9

External URL images still work

![alt](https://...) typed manually must still render. The custom marked renderer only intercepts jazz: — all other URLs pass through unchanged.

10

Image sizing inside Reveal.js slides

Reveal is 960×700px. Images need max-width: 100%; max-height: 60vh; object-fit: contain to not overflow the slide or push content off screen.

11

Blob URL memory leak

URL.createObjectURL() leaks memory if not revoked. The useJazzImages hook must call URL.revokeObjectURL() on cleanup (useEffect return).

12

SSR / server-side rendering

Next.js may server-render slides. FileStream, URL.createObjectURL, and DOM APIs are browser-only. The hook must guard with typeof window !== 'undefined' or use "use client".
