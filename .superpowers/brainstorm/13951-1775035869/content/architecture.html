Full image handling architecture

One approach — designed so every existing render site works without being rewritten.

The complete flow

✏️ Authoring (editor)

── Paste / Drop / /image ──
blob → createImage()
↓ Jazz upload
insert: ![▌](jazz:co_z123)
placeholder during upload:
![uploading…]()
→ replaced when done

🚫 Edge cases

non-image file → ignore silently
file > 5MB → resize to max 1024px
external URL typed → passes through untouched

🖥️ Rendering (everywhere)

── marked custom renderer ──
jazz:co_z123
↓ outputs
<img data-jazz-id="co_z123">
DOMPurify ✅ (data-attr is safe)
── useJazzImages hook ──
scans DOM for data-jazz-id
FileStream.toBlob() → objectURL
img.src = "blob:http://..."

📡 Presenter + Audience

LiveSession.markdown (frozen)
contains jazz:co_z123
same renderer → same hook
both see the image ✅
Jazz peer cache → works offline ✅
load fail → broken img placeholder ✅
Key insight: slideMarkdownToSafeHtml stays synchronous. The useJazzImages hook handles async resolution client-side. Zero changes to existing render call sites — just wrap the HTML container with the hook.
Scope of change:
① render-slide-markdown.ts — add custom marked renderer for jazz: URLs
② useJazzImages hook — new hook, resolves data-jazz-id attrs to blob URLs
③ DeckMarkdownEditor — add paste + drop + /image handlers (CodeMirror extensions)
④ RevealSlideBody — wrap with useJazzImages
⑤ QuestionSlideCard + PollSlideCard prompt containers — wrap with useJazzImages
