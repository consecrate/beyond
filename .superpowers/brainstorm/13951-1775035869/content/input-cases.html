All the ways an image can arrive in the editor

Before we finalize the design, let's make sure we cover every realistic authoring scenario — not just paste from clipboard.

A

Paste from clipboard

Screenshot, copied image from browser or Figma. The Obsidian-style flow — most common for teachers building slides quickly.

B

Drag & drop a file

Drag an image file from Finder/Explorer onto the editor. Common when authoring with local assets. CodeMirror already has dropCursor: true enabled.

C

External URL typed manually

Author types ![alt](https://...) with a public URL. This already works today via marked — but we need to make sure our custom renderer doesn't break it.

D

File picker button

A small "Insert image" button in the toolbar that opens a native file dialog. Good fallback for users who don't know about paste/drag.
