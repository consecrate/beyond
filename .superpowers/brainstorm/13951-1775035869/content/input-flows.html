3 ways to insert an image — all in scope for v1

Here's how each one feels in the editor. All three result in the same thing: ![alt](jazz:co_z...) inserted at the right position.

A — Paste from clipboard (Ctrl/Cmd+V)
# How photosynthesis works
| ← cursor here
Paste image here...
→
# How photosynthesis works
![image](jazz:co_z123abc)
| ← cursor moves after

Uploading... spinner shown in status bar. Inserted at cursor position.

B — Drag & drop from Finder/Explorer (smart drop)
# How photosynthesis works
← drop indicator line
The process involves...
→
# How photosynthesis works
![image](jazz:co_z456def)
The process involves...

CodeMirror's dropCursor already shows the line. We intercept the drop event, get the position, insert there.

C — /image inline command (new ✨)
# How photosynthesis works
Type /image + Enter
→
# How photosynthesis works
/image removed, file picker opens
![image](jazz:co_z789ghi)

On Enter/Space after /image: delete the text, open native file picker, insert at that line position.

All three upload via createImage(blob, { owner: me }) and insert ![alt](jazz:co_z...) at the exact position. Does the flow look right?
